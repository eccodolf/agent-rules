# Сценарии использования Qwen Code

Этот каталог содержит практические сценарии для типовых типов проектов, где Qwen Code должен работать по правилам из `.ai/`, `docs/` и `README.md`.

Цель сценариев:
- показать, как должна выглядеть целевая структура проекта;
- показать, какие `.ai`-правила нужно подключать в первую очередь;
- дать команды для получения безопасного каркаса из git в PowerShell и Linux/OEL;
- снизить риск хаотичной структуры, из-за которой агент начинает нарушать архитектурные и security-ограничения.

## Как использовать сценарии

1. Выберите сценарий, который ближе всего к вашему проекту.
2. Создайте базовую структуру папок и файлов командами из сценария.
3. Добавьте `AGENTS.md`, `.ai/` и только нужные модульные правила.
4. После этого уже используйте Qwen Code для генерации кода, тестов и инфраструктуры.

Не нужно:
- смешивать несколько сценариев без явной причины;
- включать весь `.ai/` в контекст сразу;
- пропускать системные правила безопасности и контекста.

## Общие требования ко всем сценариям

Для всех типов проектов рекомендуется:
- держать `AGENTS.md` в корне;
- держать `.ai/README.md` и нужные `.ai/modules/*` в проекте;
- разделять код, конфиг, тесты и документацию;
- не хранить секреты в репозитории;
- заранее предусматривать логирование, тесты и rollback-подход;
- задавать структуру проекта до того, как Qwen Code начнет масштабно генерировать файлы.

## Откуда брать каркас

Каркасы лежат в:
- `templates/scenarios/*` — структура конкретного проекта;
- `.ai/` — набор правил, который нужно подтянуть вместе с каркасом.
