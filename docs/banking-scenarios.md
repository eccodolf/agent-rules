# Финансовые сценарии (banking patterns)

## Назначение

Набор типовых сценариев для банковских систем.

---

## 1. Обработка платежа

Требования:
- idempotency
- audit trail
- transaction

---

## 2. Антифрод

- анализ транзакций
- риск-скоринг
- блокировка операций

---

## 3. Retry-safe операции

- повторная обработка без дублей

---

## 4. Dual control

- подтверждение двумя ролями

---

## 5. Rollback

- откат операций

---

## Вывод

Все сценарии должны быть:
- безопасны
- идемпотентны
- аудируемы
