# Правила составления промптов

## Основные принципы

1. Краткость и точность
2. Явная структура
3. Минимально достаточный контекст
4. Явные ограничения
5. Контроль формата ответа

---

## Базовая структура промпта

Рекомендуется использовать шаблон:

```
Роль
Контекст
Задача
Ограничения
Формат ответа
```

---

## Ограничение контекста

### Правила

- Не передавать весь код проекта
- Передавать только релевантные файлы/фрагменты
- Удалять:
  - логи
  - лишние комментарии
  - неиспользуемый код

### Практика

Плохо:
```
Вот весь проект (1000 строк)
```

Хорошо:
```
Вот функция и связанный код (50 строк)
```

---

## Антипаттерны

- Слишком общий запрос
- Отсутствие ограничений
- Неопределенный формат ответа
- Передача чувствительных данных

---

## Пример хорошего промпта

```
Ты senior Python разработчик.

Контекст:
Есть FastAPI сервис.

Задача:
Добавить endpoint /users.

Ограничения:
- без сторонних библиотек
- использовать pydantic

Формат ответа:
- только код
```

---

## Дополнительные рекомендации

- Делить сложные задачи на шаги
- Использовать chain-of-thought только при необходимости
- Фиксировать стиль ответа
- Использовать шаблоны
