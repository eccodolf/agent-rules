# Audit & Observability для LLM-агента

## Назначение

Контроль действий агента, воспроизводимость решений и возможность расследования.

---

## Что логировать

- тип запроса
- использованные источники
- итоговое решение
- действия (если есть tools)

---

## Что НЕ логировать

- секреты
- персональные данные
- полный prompt

---

## Метрики

- latency
- error rate
- usage
- violations

---

## Трассировка

- trace_id на каждый запрос
- связь prompt → output → действия

---

## Audit trail

- кто вызвал
- что сделал агент
- какие файлы использовал

---

## Вывод

Без audit невозможно использовать LLM в банке.
