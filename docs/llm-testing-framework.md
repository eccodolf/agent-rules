# LLM Testing Framework

## Назначение

Этот документ описывает подход к тестированию LLM и агентных сценариев.

Цель:
- обеспечить предсказуемость поведения;
- выявлять регрессии при изменении модели или промптов;
- проверять безопасность и корректность output.

---

## Уровни тестирования

### 1. Prompt unit tests

Проверяют конкретный шаблон промпта.

Проверки:
- корректность формата;
- наличие обязательных блоков (DSL);
- отсутствие запрещенных паттернов;
- стабильность структуры ответа.

---

### 2. Scenario tests

Проверяют end-to-end сценарии:
- генерация endpoint;
- генерация SQL;
- DevOps сценарии;
- banking сценарии.

---

### 3. Security tests

Проверяют:
- prompt injection устойчивость;
- отсутствие утечек;
- корректное соблюдение ограничений.

---

### 4. Regression tests

Сравнение:
- старый output vs новый output;
- после обновления модели или правил.

---

## Формат теста

```json
{
  "name": "fastapi_endpoint",
  "input": "PROMPT",
  "expected": {
    "must_contain": ["router", "pydantic"],
    "must_not_contain": ["print(", "SELECT *"]
  }
}
```

---

## Метрики

- success rate
- violation rate
- hallucination rate
- security violations

---

## Автоматизация

- запуск в CI
- проверка при изменении prompt templates
- проверка при обновлении модели

---

## Вывод

LLM без тестирования — непредсказуемый компонент.

Testing framework обязателен для production.
