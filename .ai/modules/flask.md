# Flask rules

## Основные принципы

- Использовать blueprints.
- Разделять HTTP-слой, бизнес-логику и доступ к данным.
- Не превращать route handlers в место для всей логики сразу.
- Явно задавать contracts входа и выхода.

## Безопасность

- Валидировать input.
- Не использовать debug в prod.
- Не возвращать внутренние ошибки и stack traces клиенту.
- Проверять auth/authz явно.
- Не использовать небезопасные шаблоны и HTML-markup без контроля экранирования.

## Надежность для production

- Для внешних вызовов использовать timeout и обработку ошибок.
- Для mutating actions учитывать идемпотентность и повторные запросы.
- Не хранить критичное состояние только в памяти процесса.
- Избегать глобального состояния без явного контроля.

## Что запрещено

- SQL прямо в route без необходимости;
- бизнес-логика в decorators и hooks без прозрачности;
- скрытые side effects во время сериализации ответа.
