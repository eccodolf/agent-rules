# boto / Yandex Cloud rules

## Основные принципы

- не хранить ключи в коде
- использовать IAM

## Практика

- использовать SDK безопасно
- логировать операции

## Логирование

- Логировать тип облачной операции, целевой ресурс, среду и outcome.
- Не логировать access keys, secret values, подписанные URL и содержимое секретов.
- Для mutating операций логировать инициатора, resource id и контекст отката.
- Ошибки прав доступа и аутентификации логировать без раскрытия чувствительных деталей политики.
