# Docker Compose rules

## Основные принципы

- не использовать latest
- задавать версии

## Практика

- разделять сервисы
- использовать env vars

## Логирование

- Логировать версии образов, профиль запуска и ключевые operational events для compose-окружения.
- Не писать секреты и полные значения env vars в логи, примеры и diagnostic output.
- Для multi-service сценариев обеспечивать возможность связать логи приложения, БД и очередей через correlation id или trace id, если стек это поддерживает.
