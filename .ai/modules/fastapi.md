# FastAPI module rules

## Назначение

Этот документ задает правила для задач, связанных с FastAPI.

## Основные правила

- Использовать явные схемы `pydantic` для входа и выхода.
- Не возвращать ORM-объекты напрямую из endpoint.
- Валидировать все входные параметры.
- Для ошибок использовать предсказуемые HTTP-коды и структурированные ответы.
- Разделять router, service, repository и schema слои.
- Не размещать бизнес-логику внутри endpoint-функций.
- Для зависимостей использовать `Depends`.
- Любые внешние вызовы должны иметь таймауты и обработку ошибок.
- Логирование не должно содержать секреты, токены, персональные данные и полные payload с чувствительными полями.

## Безопасность

- Проверять аутентификацию и авторизацию явно.
- Не использовать debug-конфигурации в production.
- Ограничивать выдачу внутренних деталей ошибок.
- Для загрузки файлов задавать ограничения размера и типа.
- Проверять пользовательский ввод перед использованием в SQL, filesystem и внешних запросах.

## Архитектурные рекомендации

- `api/routers/` — только HTTP-слой.
- `services/` — бизнес-логика.
- `repositories/` — доступ к данным.
- `schemas/` — входные и выходные модели.
- `dependencies/` — DI и security dependencies.

## Что запрещено

- Писать SQL прямо в endpoint.
- Смешивать transport-model и domain-model без необходимости.
- Использовать глобальное состояние без явного контроля.
- Возвращать stack trace клиенту.

## Примеры промптов

### Простой

```text
Ты senior Python backend разработчик.

Контекст:
Есть FastAPI сервис.

Задача:
Добавь GET endpoint `/health`.

Ограничения:
- вернуть JSON со статусом `ok`
- без сторонних библиотек

Формат ответа:
- только код
```

### Средний

```text
Ты senior Python backend разработчик.

Контекст:
Есть FastAPI сервис с разделением на router/service/repository.
Нужно добавить CRUD для сущности customer.

Задача:
Сгенерируй схемы, router и service слой для создания и получения customer.

Ограничения:
- использовать pydantic
- не размещать бизнес-логику в router
- ошибки вернуть через HTTPException
- добавить примеры response model

Формат ответа:
1. структура файлов
2. код по файлам
```

### Сложный

```text
Ты senior backend архитектор по FastAPI.

Контекст:
Есть банковский сервис обработки заявок. Нужен новый endpoint для приема заявки, асинхронной постановки в Celery и аудита действий.

Задача:
Спроектируй и реализуй HTTP endpoint, service слой и контракт ответа.

Ограничения:
- FastAPI
- pydantic v2
- idempotency key в header
- audit logging обязателен
- чувствительные поля маскировать в логах
- endpoint не должен напрямую работать с Celery broker
- предусмотреть коды ошибок 400/401/403/409/500
- не использовать глобальные переменные

Формат ответа:
1. краткое решение
2. структура модулей
3. код
4. список рисков и проверок
```
