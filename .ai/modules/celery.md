# Celery module rules

## Основные правила

- Все задачи должны быть идемпотентными.
- Использовать retry с backoff.
- Ограничивать время выполнения задач.
- Явно разделять task orchestration и бизнес-логику.

## Безопасность

- Не передавать секреты в payload задачи.
- Не логировать чувствительные данные.

## Примеры промптов

### Простой

```text
Напиши простую Celery задачу.
```

### Средний

```text
Сгенерируй Celery task с retry и логированием.
```

### Сложный

```text
Ты backend разработчик банковской системы.

Контекст:
Асинхронная обработка платежей.

Задача:
Спроектируй Celery pipeline.

Ограничения:
- идемпотентность
- retry
- audit

Формат:
- архитектура
- код
```
