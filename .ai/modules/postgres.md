# PostgreSQL module rules

## Основные правила

- Использовать параметризованные запросы
- Не использовать SELECT *
- Явно указывать список колонок
- Использовать транзакции для критичных операций
- Ограничивать выборки (LIMIT)
- Использовать индексы осознанно

## Безопасность

- Запрещены динамические SQL без параметров
- Проверять input перед передачей в запрос
- Не логировать raw SQL с параметрами

## Примеры промптов

### Простой

```text
Ты эксперт по PostgreSQL.

Задача:
Написать SQL для получения пользователя по id.

Ограничения:
- без SELECT *

Формат:
- SQL
```

### Средний

```text
Ты эксперт по PostgreSQL.

Контекст:
Есть таблицы users и orders.

Задача:
Получить список пользователей с количеством заказов.

Ограничения:
- использовать JOIN
- добавить alias

Формат:
- SQL
```

### Сложный

```text
Ты эксперт по PostgreSQL в банковской системе.

Контекст:
Есть таблица транзакций.

Задача:
Написать безопасный запрос агрегации по операциям.

Ограничения:
- учитывать транзакционность
- исключить race condition
- добавить индексы

Формат:
- SQL
- пояснение
```
