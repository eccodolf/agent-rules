# PostgreSQL module rules

## Основные правила

- Использовать параметризованные запросы.
- Не использовать `SELECT *`.
- Явно указывать список колонок.
- Использовать транзакции для критичных операций.
- Ограничивать выборки и пагинировать потенциально большие результаты.
- Использовать индексы осознанно.

## Безопасность

- Запрещены динамические SQL без параметров.
- Проверять input перед передачей в запрос.
- Не логировать raw SQL с параметрами.
- Не смешивать доступ к БД и пользовательский ввод без валидации и нормализации.

## Надежность для production

- Понимать уровень изоляции и границы транзакции.
- Для денежных и stateful операций учитывать duplicate processing и race conditions.
- Для миграций учитывать lock impact и совместимость со старой версией кода.
- Для тяжелых запросов думать про explain/analyze, даже если это не показывается в финальном ответе.

## Практика

- Для write-операций явно описывать ключ идемпотентности или уникальности, если он критичен.
- Для агрегаций и batch-processing ограничивать размер окна.
- Для schema changes предусматривать phased rollout там, где это нужно.

## Что запрещено

- destructive update/delete без фильтра;
- implicit ordering без `ORDER BY`;
- миграции, которые в один шаг ломают обратную совместимость;
- запросы, полагающиеся на "малый объем данных" без оговорки.
