# Infrastructure rules

## Назначение

Правила генерации инфраструктуры.

## Основные требования

- Все изменения должны быть обратимыми.
- Использовать declarative подход.
- Не использовать latest теги.

## Безопасность

- Не отключать TLS и auth.
- Не раскрывать секреты.

## Практика

- использовать versioning
- применять rollback стратегии
