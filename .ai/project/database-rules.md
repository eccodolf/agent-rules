# Database rules

## Назначение

Правила работы с БД при генерации кода, SQL и миграций.

## Основные требования

- Использовать параметризованные запросы.
- Избегать `SELECT *`.
- Явно указывать список колонок.
- Учитывать индексы и стоимость запросов.
- Писать идемпотентные операции там, где возможен повторный вызов.

## Безопасность

- Не хардкодить credentials.
- Валидировать входные данные до передачи в запрос.
- Не использовать string interpolation для SQL.
- Не логировать raw SQL с чувствительными параметрами.

## Надежность для production

- Понимать transaction boundaries.
- Избегать неограниченных update/delete операций.
- Не делать schema changes без rollback-плана.
- Проверять, как запрос поведет себя на больших объемах данных.
- Разделять read и write path, если это влияет на блокировки и latency.

## Практика

- Использовать ORM или query builder, если это уже принято в проекте.
- Для сложных запросов добавлять пояснение по индексам и ограничениям.
- Для массовых изменений указывать batch strategy.
- Для миграций учитывать lock impact и backward compatibility.

## Что запрещено

- DDL/DML без понимания последствий;
- запросы, зависящие от неявного порядка строк;
- destructive maintenance без backup/rollback considerations;
- schema migration, которая ломает старую версию приложения без переходного периода.
