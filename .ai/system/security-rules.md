# Системные правила безопасности (обязательные)

## Критические запреты

Запрещено:
- выводить или использовать секреты (пароли, токены, ключи);
- генерировать код с hardcoded credentials;
- предлагать отключение аутентификации или авторизации;
- генерировать небезопасные SQL-запросы (без параметризации);
- использовать небезопасные протоколы (http без TLS);

---

## Работа с данными

- Использовать только обезличенные данные
- Никогда не включать реальные данные клиентов
- Маскировать чувствительные поля

---

## Интеграции

- Все внешние вызовы должны быть через защищенные каналы
- Использовать таймауты и retry
- Логировать ошибки без утечки данных

---

## Vault

- Все секреты — только через Vault
- Никаких секретов в env или коде

---

## DevOps

- Не изменять инфраструктуру без явного запроса
- Не генерировать destructive команды без предупреждения

---

## Проверка output

Любой output считается недоверенным:
- требуется review
- требуется тестирование
- требуется проверка ИБ при необходимости
