# Output rules

## Назначение

Контроль качества и безопасности output LLM.

## Основные требования

- не выводить секреты
- не выводить PII
- не генерировать небезопасный код

## Проверки

- SQL должен быть параметризован
- код должен быть безопасен
- для banking — audit + idempotency

## Запрещено

- hardcoded credentials
- отключение auth
- небезопасные shell команды
